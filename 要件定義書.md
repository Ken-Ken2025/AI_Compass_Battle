# 料理レシピWebアプリ 要件定義書

**プロジェクト名**: AI Compass Dev Battle — 料理レシピWebアプリ  
**作成日**: 2026-05-22  
**バージョン**: 1.0.0  
**ステータス**: ドラフト

---

## 1. プロジェクト概要

### 1.1 目的

本プロジェクトは、ユーザーが保有する食材を入力するだけでAIが最適なレシピを提案・生成するWebアプリケーションを開発することを目的とする。  
あわせて、チームでGit / GitHub を用いた共同開発の実践練習の場とする。

### 1.2 背景

- 「今ある食材で何が作れるか分からない」という日常的な課題を解決する
- AIを活用することで、ユーザーの食材・嗜好・栄養バランスに合わせた個別提案を実現する

### 1.3 スコープ

| 対象 | 内容 |
|------|------|
| 対象ユーザー | 自炊をする個人ユーザー全般（10代〜60代） |
| 対象プラットフォーム | Webブラウザ（PC・スマートフォン対応） |
| 対象外 | ネイティブアプリ、ECサイト連携、食材発注機能 |

## 2. 機能要件

### 2.1 機能一覧

| ID | 機能名 | 優先度 | 概要 |
|----|--------|--------|------|
| F-01 | 食材入力 | 必須 | ユーザーが手持ちの食材を入力する |
| F-02 | AIレシピ提案 | 必須 | 入力食材をもとにAIがレシピを生成・提案する |
| F-03 | レシピ表示 | 必須 | 提案されたレシピの詳細（材料・手順）を表示する |
| F-04 | レシピ検索 | 推奨 | キーワード・カテゴリ・調理時間でレシピを絞り込む |
| F-05 | お気に入り管理 | 推奨 | 気に入ったレシピをブックマーク・一覧表示する |
| F-06 | ユーザー認証 | 推奨 | アカウント登録・ログイン・プロフィール管理 |
| F-07 | レシピ投稿 | 推奨 | ユーザー自身がオリジナルレシピを登録・共有する |
| F-08 | 栄養情報表示 | 任意 | 各レシピのカロリー・栄養バランスを表示する |
| F-09 | 買い物リスト生成 | 任意 | 不足食材を自動でリスト化する |

### 2.2 主要機能詳細

#### F-01: 食材入力

- テキストボックスへの自由入力（複数食材をカンマ区切りまたは改行で入力）
- 食材タグのオートコンプリート機能（入力補完）
- アレルギー・除外食材の設定

#### F-02: AIレシピ提案

- 入力された食材をプロンプトとして Claude API に送信
- 提案レシピ数：デフォルト3件（最大5件まで変更可）
- 提案条件のカスタマイズ：
  - 調理時間の上限（例：30分以内）
  - 難易度（簡単・普通・本格的）
  - 料理ジャンル（和食・洋食・中華・その他）
  - カロリー目安
- AIレスポンスはストリーミング表示でリアルタイムに出力する

#### F-03: レシピ表示

- レシピ名・アイキャッチ画像
- 材料リスト（人数分の量を自動換算）
- 手順（ステップ形式、画像付きオプション）
- 調理時間・難易度・カロリーの概要タグ
- 関連レシピの横断表示

#### F-04: レシピ検索

- キーワード全文検索
- カテゴリフィルター（和食・洋食・中華・スイーツ等）
- 調理時間フィルター（〜15分 / 〜30分 / 〜60分 / 60分超）
- 難易度フィルター
- 並び替え（新着順・人気順・調理時間順）

## 3. 非機能要件

### 3.1 パフォーマンス

| 指標 | 目標値 |
|------|--------|
| ページ初回読み込み | 3秒以内（LCP） |
| AIレシピ生成開始（ストリーミング開始） | 入力後2秒以内 |
| 検索レスポンス | 1秒以内 |

### 3.2 可用性

- 目標稼働率：99.5%以上
- メンテナンス時間帯：毎週日曜 02:00〜04:00（計画停止）

### 3.3 セキュリティ

- HTTPS 通信必須
- APIキーはサーバーサイドで管理（フロントエンドに露出させない）
- ユーザー入力のサニタイズ・バリデーション
- プロンプトインジェクション対策（AI入力の検証）

### 3.4 ユーザビリティ

- スマートフォン対応（レスポンシブデザイン）
- WCAG 2.1 AA レベルのアクセシビリティ準拠
- 日本語UIのみ（多言語対応はスコープ外）

### 3.5 スケーラビリティ

- 初期想定同時接続数：100ユーザー
- 将来的に1,000ユーザーまでスケールアウト可能な設計

## 4. システム構成

### 4.1 技術スタック（案）

| レイヤー | 技術 |
|---------|------|
| フロントエンド | React + TypeScript + Tailwind CSS |
| バックエンド | Node.js + Express（または Next.js API Routes） |
| AI | Claude API（Anthropic SDK） |
| データベース | PostgreSQL（レシピ・ユーザー管理） |
| ストレージ | Amazon S3 / Cloudflare R2（画像） |
| ホスティング | Vercel / Render |
| 認証 | NextAuth.js / Clerk |

### 4.2 システム構成図（概略）

```
[ ユーザー（ブラウザ） ]
        ↓
[ フロントエンド (React / Next.js) ]
        ↓
[ バックエンド API ]
    ├── [ Claude API ] ← AIレシピ生成
    ├── [ PostgreSQL ]  ← レシピ・ユーザーデータ
    └── [ S3 / R2 ]     ← 画像ストレージ
```

## 5. データ設計（主要エンティティ）

### 5.1 Recipe（レシピ）

| フィールド | 型 | 説明 |
|-----------|-----|------|
| id | UUID | 主キー |
| title | string | レシピ名 |
| ingredients | JSON | 材料リスト |
| steps | JSON | 手順リスト |
| category | enum | 料理ジャンル |
| cooking_time | integer | 調理時間（分） |
| difficulty | enum | 難易度 |
| calories | integer | カロリー（kcal） |
| is_ai_generated | boolean | AI生成フラグ |
| created_by | UUID | 作成者（userID） |
| created_at | timestamp | 作成日時 |

### 5.2 User（ユーザー）

| フィールド | 型 | 説明 |
|-----------|-----|------|
| id | UUID | 主キー |
| email | string | メールアドレス |
| display_name | string | 表示名 |
| allergies | JSON | アレルギー情報 |
| created_at | timestamp | 登録日時 |

## 7. AIプロンプト設計方針

### 7.1 基本プロンプト構造

```
システム: あなたは優秀な料理研究家です。与えられた食材と条件を最大限活用して、
          実用的で美味しいレシピを提案してください。

ユーザー: 
【手持ち食材】{食材リスト}
【調理時間の上限】{時間}分以内
【難易度】{難易度}
【料理ジャンル】{ジャンル}
【その他条件】{カロリー制限・アレルギー等}

{件数}件のレシピを提案してください。
各レシピはJSON形式で返答してください。
```

### 7.2 出力フォーマット

```json
{
  "recipes": [
    {
      "title": "レシピ名",
      "ingredients": [{"name": "食材名", "amount": "量"}],
      "steps": ["手順1", "手順2"],
      "cooking_time": 30,
      "difficulty": "普通",
      "calories": 450,
      "tips": "コツや注意点"
    }
  ]
}
```

## 9. リスクと対策

| リスク | 影響度 | 対策 |
|--------|--------|------|
| Claude API のレートリミット超過 | 高 | リクエストキューイング・キャッシュ活用 |
| AI生成レシピの品質ばらつき | 中 | プロンプトチューニング・ユーザーフィードバック収集 |
| 食材名の表記ゆれによる認識誤り | 中 | 入力正規化・オートコンプリートで補正 |
| スパム投稿・不適切コンテンツ | 中 | レート制限・モデレーション機能の実装 |
